## 1. 핵심 철학

**가치는 회원 명부에. 구조는 최대한 단순하게. 개인정보·보안은 제대로.**

- 현실: 회원조차 거의 안 보는 사이트
- 유일한 핵심 가치: **회원 명부** (사진·직업·연락처로 서로 누군지 알게 됨)
- 회원제로 가되, 로그인 상태 유지로 체감 마찰은 0에 가깝게

---

## 2. 운영 모델 (확정)

### 회원 인증 = 휴대폰번호 가입
- **아이디 = 휴대폰번호** (별도 아이디 없음)
- 가입: 휴대폰번호 + 비밀번호(2회 입력) + **문자 컨펌코드** 입력 → 완료
- **로그인 상태 유지**(persistent session) → 그 뒤로 매번 로그인 불필요
- 비번 분실 → 휴대폰번호로 **문자 재설정**
- 가입 시 개인정보 동의 = **본인이 직접 체크** (가장 강한 동의 증거)

### 가입 → 승인 흐름
```
가입(문자인증 완료)  →  지회 관리자 승인  →  정식 회원(명부 노출·게시판 작성)
```
- 문자인증 = "진짜 번호" 확인 / 관리자 승인 = "우리 지회 사람" 확인
- 외부인 무단 가입·열람 차단

### 권한 계층
| 주체 | 인증 | 권한 |
|---|---|---|
| 회원 | 휴대폰 가입 + 승인 | 명부 열람, 게시판 작성, 본인 정보 수정 |
| 지회 관리자 | 관리자 계정(Auth + RLS) | 콘텐츠·회원·가입승인·회계·지회설정 |
| 본부(중앙) | 슈퍼관리자 | 지회 생성/삭제, 관리자 배정 |

## 3. 기능 구성

### 회원 화면 (로그인 필요)
| 기능 | 상태 | 비고 |
|---|---|---|
| 👤 **회원 명부** (사진·직업·연락처) | 있음 → **메인 승격** | 사진 크게, 직업 필터·검색, 모바일 최적화 |
| 📝 게시판 (행사후기) | **신규** | 로그인 회원 누구나 작성, 실명 계정 연동 |
| 🖼️ 사진 / 📅 행사 / 📢 공지 | 있음 | 유지 |
| ⚙️ 내 정보 수정 / 전화 공개·비공개 | 신규 | 본인이 직접 제어 |

### 공개 화면 (비로그인)
- 지회 소개 최소 정보
- ✍️ **가입 폼** (휴대폰 인증 + 동의)
- 📄 **개인정보처리방침** (지회명만 치환하는 공용 템플릿)

### 관리자 화면
- **회원 관리**: 가입 승인 / 수정 / 탈퇴
- 공지 · 행사 · 사진 · 게시판 관리(불량글 삭제)
- 💰 **회계**: 숫자만 입력 → 잔액·총계 자동 계산, 안 쓰는 지회는 메뉴 숨김
- ⚙️ **지회 설정**: 로고 · 색상 · 연락처 · SNS

### 제거
- 소식(news) → 공지로 통합
- 자료실(documents), 소개/resources 등 미사용 페이지

## 4. 개인정보 처리

- **동의**: 가입 시 본인이 직접 체크 (수집항목·목적·명부공개·철회권 고지)
- **전화번호 비공개 토글**: 회원 본인이 명부 노출 여부 선택
- **처리방침 페이지** 필수 제공 (공용 템플릿)
- 주민번호 등 민감정보 미수집
- 탈퇴 시 즉시 삭제, 개인별 차단 가능(공용비번 약점 해소)

## 5. 구축 단계

### Phase 0 — 데이터 모델 확정
- 누락 테이블 추가: `members`, `events`, `posts`(게시판), `transactions`
- `branches`에 `youtube_url`·`facebook_url`·`instagram_url` 추가
- 회원 테이블: 휴대폰(=로그인) · 비번 · 승인상태 · 동의일 · 전화공개여부
- news 카테고리 정리(공지로 통합)
- 회원 `category`(가입유형) ↔ `industry`(직업) 의미 분리

### Phase 1 — Supabase 실연동
- `mockData.ts` → 실제 DB 쿼리로 교체
- 관리자 CRUD 실제 insert/update/delete 연결
- 사진: Supabase Storage 연동(+ 업로드 용량/리사이즈 처리)

### Phase 2 — 인증
- **회원**: Supabase Auth 휴대폰 OTP 가입 + 비번 + 세션 유지
  - SMS 발송 서비스 연동(알리고/NHN 등) + 발신번호 등록
- **관리자**: 진짜 Auth + RLS로 지회별 데이터 격리
- 가입 → 승인 대기 → 관리자 승인 흐름
- 로그인 시 배정 지회로 라우팅 (yeongdong 고정 제거)

### Phase 3 — 기능 정리·신규
- 회원 명부 강화 (사진·필터·검색·모바일)
- 게시판(행사후기) 신규
- 회계 자동계산(과거 항목 수정 시 이후 잔액 재계산 포함) + 미사용 지회 숨김
- 지회 설정 화면 / 내 정보 수정 / 처리방침 페이지
- 기존 불일치 정리 (가입신청 메뉴 누락, 회원분류 꼬임, 비공개글 관리 등)

### Phase 4 — 배포
- Vercel 배포 + 도메인 연결
- 영동 기존 실데이터 이관 + 백업 체계
- 신규 지회 추가 절차 문서화 (중앙에서 지회 생성 + 관리자 배정)

## 7. 발견된 기존 불일치 (Phase 3에서 정리)
1. 가입신청 관리 페이지가 사이드바에 없어 접근 불가
2. 회원 분류(category) 값이 실제 데이터와 불일치 (가입유형 vs 직업 혼재)
3. 소식 관리에서 비공개 글이 목록에 안 뜸 (`getBranchNews` published 필터)
4. 로그인이 항상 yeongdong으로 고정 이동
5. Supabase 스키마가 앱 타입보다 미완 (members/events/documents 등 테이블 부재)
